# 安全策略

## 支持范围

本仓库为 **纯静态前端游戏**（HTML/CSS/JS），无服务端、无用户账户体系。若你发现与 **本仓库代码** 相关的安全问题（例如可被恶意页面利用的脚本行为），欢迎报告。

## 如何报告

1. 请 **不要** 在公开 Issue 中披露可利用细节  
2. 若仓库开启了 **Security advisories / Private vulnerability reporting**，请优先通过该渠道提交  
3. 否则，可向仓库维护者发送私信说明（具体联系方式以 GitHub 主页为准）

请提供：问题类型、复现步骤、受影响版本或提交哈希（如适用）。

## 预期响应

维护者会在方便时审阅报告；小型静态项目不设 SLA，但会认真对待合理报告。
